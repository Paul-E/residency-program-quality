How noisy is the residency-program pass-rate rule?

The rule, and why size matters

Every neurosurgery residency program is judged the same way: residents must maintain at least an 80% board-pass rate, averaged over the most recent three years. The catch is that programs are not the same size. Some graduate one resident a year; a few graduate four. So the number of graduates the rule actually looks at — the denominator — ranges from about 3 to about 12.

When you judge a coin by a handful of flips, luck dominates. The same is true here: with only a few graduates, a single unlucky board failure can flip a good program from “pass” to “fail.” This page lets you change the inputs and watch how much of the outcome is driven by chance rather than by training quality.

1 · The same rule isn’t the same rule

You can’t pass “80% of 3 residents.” You either pass 2 of 3 (67%) or 3 of 3 (100%) — there is nothing in between. So an 80% rule quietly becomes a 100% rule for the smallest programs. The table below shows the real bar each program size has to clear.

Notice the 3-residents/year row: because 80% of 9 is 7.2, the program must get 8 of 9 — an 88.9% effective bar, harsher than the 2/year and 4/year programs. The rule is uneven in a way that has nothing to do with quality.

2 · What are the odds of passing, really?

Suppose we knew a program’s true quality — the long-run probability that any one of its residents passes the boards. How often would that program actually clear the rule in a given window? The answer, below, is unsettling: a program whose true quality is exactly 80% only passes the 80% rule roughly 44–66% of the time, depending on its size. Half of genuinely-adequate small programs fail by luck.

Each cell is the probability a program of that size passes the rule, given its true resident pass-rate. Greener = more likely to pass.

The curves above are “operating characteristic” curves: the probability of passing the rule (y) versus the program’s true quality (x). A perfect rule would be a straight vertical line at the standard. These gentle S-curves show how much luck is in play — and how the small programs’ curves sit lowest.

Show the math

If each of n graduates passes independently with true probability p, the number who pass is Binomial(n, p). The program clears the rule when at least k_min = ⌈cutoff × n⌉ pass, so

P(pass) = Σk = k_min..n C(n,k) pk (1−p)n−k = 1 − CDF(k_min − 1).

3 · How much can one window really tell us?

Turn it around: given what a program actually did, how sure can we be about its true quality? A confidence interval answers that. Even a “perfect” program that goes 3-for-3 has a true pass-rate that could plausibly be anywhere from about 44% to 100%. In other words, a perfect small program is statistically hard to tell apart from one whose true rate is well below the standard.

Each bar is the Wilson confidence interval for the true pass-rate, and the black dot is what the program actually scored. The dashed line is the 80% standard — wherever a bar crosses it, the data can’t confidently say the program is above or below standard. The top four rows are fixed reference cases: the smallest passing result for each program size (1–4 residents/year over 3 years). The bottom row (“your selection”) follows the Passes and Graduates sliders above, so you can test any result yourself.

Show the math & caveats

These are Wilson score intervals — the recommended choice for small samples and proportions near 100% (NIST e-Handbook 7.2.4.1; Wikipedia, “Binomial proportion confidence interval”). A 3/3 result genuinely has an upper bound of exactly 100%. All of these intervals assume each graduate’s board result is independent with a common true pass-rate.

4 · The funnel plot

This is the standard picture used in hospital-quality monitoring. Each dot is a program: its size on the horizontal axis, its observed pass-rate on the vertical. The funnel-shaped bands show the range of results we’d expect from chance alone if every program were exactly at the standard. The bands flare out for small programs — that wide scatter is normal, not evidence of bad training.

Inner band = 95% limits; outer band = 99.8% limits (the usual outlier line). Even when every program here is identical in true quality, small ones scatter far — some past the bands — purely by luck.

A point outside the funnel is a candidate for a closer look, not a verdict. And if programs genuinely differ in quality (they do), extra scatter beyond the bands reflects real differences — so strict limits can over-flag. (Spiegelhalter 2005; Verburg et al.)

5 · The real tension: noise vs. speed

The obvious fix — use a longer window — cuts the noise but slows down the discovery of a program that has genuinely started training residents poorly. We want to protect patients by catching real problems quickly, and avoid falsely condemning an adequate program that hit a rough patch. A fixed 3-year cliff does neither job well.

A better idea borrowed from manufacturing and surgical-outcomes monitoring is to accumulate evidence continuously and raise a flag only when the evidence is strong. That is what a CUSUM chart does.

6 · CUSUM monitoring with a V-mask

A CUSUM (“cumulative sum”) chart adds up how far each resident’s result falls above or below the acceptable rate. As long as a program performs adequately, the line drifts flat or upward. If quality slips, the line bends steadily downward. A V-mask — the sideways “V” laid over the most recent point — signals a problem the moment a past point pokes outside its arm. It catches a real decline faster than waiting three years, while controlling false alarms.

Acceptable rate (p0)
The pass rate you consider fine — the standard a program is meant to meet (80%). A program truly performing at this level should not trigger the alarm.
Unacceptable rate (p1)
A clearly-too-low pass rate you want to catch quickly (e.g. 65%). The chart continuously weighs each program’s results between these two levels — “does this look more like an 80% program or a 65% one?” — and signals once the evidence points to p1. The gap between p0 and p1 sets the sensitivity: a smaller gap catches gentler declines (but risks more false alarms); a larger gap reacts only to bigger collapses.
Decision interval (h)
How much accumulated evidence is required before sounding the alarm — the depth of the V-mask. A higher h demands stronger proof, so there are fewer false alarms but slower detection; a lower h reacts sooner but cries wolf more often. Values around 4–5 are the usual rule of thumb.

Each step is one graduating resident over a 7-year program (about 1–2 per year). The V-mask arms have slope ±k; any point that escapes an arm is flagged red. The deterioration scenario typically trips the alarm a year or two after quality drops — well before a 3-year average would.

Show the math & the rigorous version

We standardize each result as z = (x − p0)/σ with σ = √(p0(1−p0)), plot the running sum, and run the equivalent tabular test S_lo = max(0, S_lo − z − k), flagging when S_lo > h. The shift to detect is δ = (p0 − p1)/σ and k = δ/2 (NIST e-Handbook 6.3.2.3). For binary data the fully rigorous tool is the Bernoulli risk-adjusted CUSUM (Steiner et al., 2000), which accumulates a log-likelihood-ratio score and also accommodates differing case difficulty; the normal-theory version shown here is a close, easier-to-picture approximation.

The bottom line

The current rule does not evaluate all programs with equal precision. Over only three years, observed 80% compliance carries a lot of random noise — it can fail genuinely-adequate small programs and pass genuinely-weak ones by chance. A fairer system accounts for size and uncertainty: report confidence intervals, use funnel limits instead of one flat line, and monitor continuously so real problems are caught quickly without condemning programs for bad luck.
